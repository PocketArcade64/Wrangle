[M1.7] 2026-07-06 - HOTFIX: black screen = CSV column misalignment
----------------------------------------------------------------------
- User reported black screen on Mac. Verified their GitHub repo had all
  files current (src/, public/sprites/ 48 files, ui/button.ts, latest
  species.ts + BootScene). Root cause was in MY M1.6 creatures.csv:
  header has 24 columns but every hand-written data row had only 23
  fields (one comma short), shifting textureKey values like
  "Herbifuzz_80x80" into the numeric dashIntervalMs column ->
  buildSpecies() throws "not a number" at module load -> nothing ever
  renders -> black screen.
- Regenerated the CSV with a script (not by hand) and VERIFIED with awk
  that all 46 lines have exactly 24 fields and values land in the right
  columns.
- LESSON (added to OPUS_TIPS): never hand-count CSV commas; generate
  rows programmatically and verify field counts with awk before
  shipping. A data file error crashes at module load = black screen
  with no visible error unless the browser console is open.
- STATUS: user to re-upload src/data/creatures.csv and re-test.
