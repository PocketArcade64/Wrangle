[M1.3] 2026-07-06 - Whole-bar damage + creatures.csv spreadsheet
----------------------------------------------------------------------
Playtest feedback round 3 (decay pacing approved):
- HEALTH is now counted in whole bars (5). Attacks remove whole bars
  only - no partial segment fills. attackDamage in creature data now
  means "bars per hit" (dustdevil: 1).
- NEW: creature data moved from TS literals to src/data/creatures.csv,
  a real spreadsheet the user edits in Excel/Numbers/Sheets. Vite
  imports it with ?raw; species.ts now parses + validates it at build
  time (descriptive per-row errors name the creature + column). Added
  src/vite-env.d.ts for the ?raw import type. New columns: type,
  atkStyle (stored now, used by M3 battles).
- docs/CREATURES.md reworked into the how-to-edit-the-CSV guide with
  full column reference + sprite gallery section.
- NOTE: species.ts had a UTF-8 mangling incident via PowerShell
  Get-Content/Set-Content (em-dashes corrupted); rewritten clean with
  plain hyphens. Lesson: never round-trip project files through
  PS 5.1 default encoding - use the Write tool.
- STATUS: awaiting playtest round 4. User can now try adding a test
  creature row to creatures.csv themselves.

[M1.2] 2026-07-06 - Gauge decay replaces stun; HEALTH; no rope budget
----------------------------------------------------------------------
Playtest feedback round 2 ("new ranger line works great!"):
- REMOVED the dazed/stun mechanic entirely (release-mid-draw now just
  drops the line, no penalty; CreatureActor stun state + riled speedup
  deleted).
- NEW anti-stall pressure: if 4s pass without closing a loop around the
  creature, the capture gauge drains quickly (GAUGE_DECAY_DELAY_S=4,
  GAUGE_DECAY_PER_S=0.75 loops/s; gaugeProgress is now a float).
- RENAMED GRIT -> HEALTH everywhere (UI label, toasts, docs).
- Lasso line thicker: LINE_WIDTH 5 -> 7.
- REMOVED the rope length meter + budget mechanic (LassoLine now gets
  Infinity budget; may return later as an upgrade axis). Bottom strip
  now shows only the segmented HEALTH bar; arena grew taller.
- STATUS: awaiting playtest round 3 (decay pacing: is 4s + 0.75/s fair
  on PECKSY/DUSTDEVIL?).

----------------------------------------------------------------------
[M1.1] 2026-07-06 - iPhone fit + Ranger-accurate line + UI rework
----------------------------------------------------------------------
Playtest feedback round 1 (tested on Mac browser + iPhone Safari):
- FIXED iPhone sizing: black bar top / bottom cut off. index.html now
  uses 100dvh (Safari's collapsing chrome broke 100vh) + viewport-fit=
  cover; main.ts computes logical height from device aspect (720 wide,
  1280-1600 tall) so tall phones fill the screen. Scenes now derive
  layout from this.scale.height instead of hardcoded 1280.
- FIXED loop rule to match Pokemon Ranger Shadows of Almia: closing a
  loop consumes ONLY the loop portion; the pre-loop tail stays on the
  field (loopable, breakable), drawing continues from the crossing
  point, loop length is refunded to the rope budget (LassoLine.ts).
- UI REWORK per user: removed the loop pips (top-left orange circles -
  user found them confusing); capture progress is now a gauge that
  follows below the creature, +10 floats up per loop, full gauge =
  captured. GRIT bar is now SEGMENTED (5 segments of 20) and moved to
  the bottom strip next to the rope meter.
- STATUS: awaiting playtest round 2 on iPhone (fit + loop feel).

[M1] 2026-07-05 - Lasso capture prototype
----------------------------------------------------------------------
- Built the full Pokemon Ranger-style capture mini-game (CaptureScene):
  loop drawing, rope budget (resets per loop), banked-loop pips,
  body-touch line break (no damage), telegraphed radial attacks that

  anti-cheese (no progress during stun, creature gets faster after).

- Docs: docs/GAME_DESIGN.md (full concept), docs/ROADMAP.md (M0-M10).
