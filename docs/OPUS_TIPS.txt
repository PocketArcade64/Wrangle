TIPS FOR CLAUDE (written by Fable 5, handing off to Opus 4.8)
=======================================================================
Hard-won lessons from building Wrangle M0 through M4. Read DEVLOG.txt
and CLAUDE.md FIRST, then this file. Everything below was paid for with
a real playtest round trip - do not relearn it the expensive way.

THE #1 CONSTRAINT: YOU CANNOT RUN THE CODE HERE
- This Windows machine has no Node.js and cannot get it. No compile, no
  dev server, no browser. Every error ships to the user's Mac and costs
  a full upload/pull/playtest round trip.
- Therefore MENTALLY COMPILE EVERYTHING before finishing: all imports
  used (tsconfig: strict + noUnusedLocals + noUnusedParameters - an
  unused import is a BUILD FAILURE), types line up, every referenced
  method exists, object literals satisfy their interfaces completely.
  Sweep every file you touched at the end of the iteration.
- Prefix intentionally-unused params with underscore (_time).
- Vite dev mode does NOT typecheck - errors surface at build. Do not
  assume "it ran for the user" means "it compiles clean".

PHASER GOTCHAS THAT COST US PLAYTEST ROUNDS (all real bugs)
- setScrollFactor(0) UI over a scrolled camera: renders right, but
  input hit-testing does NOT compensate (especially containers). The
  "BACK TO MAP button does nothing" bug. THE FIX (now the architecture
  of StageScene): scroll a WORLD CONTAINER (world.y = -scroll), keep
  the camera fixed, and make all UI plain screen-space objects. Do not
  reintroduce scrollFactor UI.
- Double centering: #game centers the canvas via CSS flex AND Phaser
  CENTER_BOTH centered via canvas margins -> flex centers the margin
  box -> intermittent down/right shift on iOS standalone launches.
  Exactly ONE centerer: autoCenter stays NO_CENTER, CSS flex owns it.
- iOS standalone (ADD TO HOME SCREEN): #game is position:fixed with
  top/bottom env(safe-area-inset-*) and the status-bar meta is
  "default" (webview starts below the Dynamic Island). Avoid
  calc()+env() and margin tricks (Safari parser quirks - failed once).
  main.ts measures the #game container and re-refreshes on a spread of
  timers (insets settle late with NO event). Testing: DELETE and re-add
  the home-screen icon; iOS caches hard.
- Scene transitions on pointerUP, never pointerdown - otherwise the
  same tap's release lands on freshly created buttons in the next
  scene (the title double-press bug).
- Scene instances persist across restart()/start(): field initializers
  run ONCE. Reset ALL mutable round state in init(). Every scene here
  follows that pattern - keep it.
- Scene-level input fires on taps over buttons too - guard with screen
  regions (see StageScene swipe vs button row).
- Scene sleep/wake is how mid-stage captures work: Stage does
  scene.run('Capture', data) + bringToTop + this.scene.sleep();
  Capture exits with scene.wake('Stage') + scene.stop(). A sleeping
  scene's clock pauses (cooldowns freeze for free). WAKE event handler
  in Stage resumes music / pending clears.
- Webfonts: document.fonts.load with a timeout race in Boot.
- Modals: scene-level drag handlers still run under a modal dim
  (known quirk, listed as polish debt).

PIXEL-ART DISCIPLINE (the user notices - these were bug reports)
- INTEGER sprite scales at INTEGER coordinates only. Fractional scale
  (1.8) made the same badge render differently per screen column
  ("thick W" artifact). Badges are x2/x3; if spacing math forces a
  fractional scale, change the layout, not the scale.
- Generated pixel art: draw per-pixel with 1px fillRect loops (see
  typeDots.ts), NOT fillCircle/antialiased shapes. Quantize decorative
  art to 4/8/16px grids (see StageScene walls: 8px staircase edges).
- Design canon (GAME_DESIGN.md "UI design system") is LAW: parchment/
  saddle/clay(the one accent)/sage/denim/ink/brass(currency only) +
  adobeRed (destructive confirms only) + wantedRed (poster ink only).
  NO rounded UI corners/gradients/glow. Text floor ~18-20px logical
  (16px tertiary labels only). Silkscreen UI, Pixelify display.
- UI conventions in force: 110px top bars, BACK at (84,55) everywhere,
  shared gold counter (ui/goldCounter.ts), wood frame language (home
  diorama + map), modal pattern = dim(depth60) + drawPixelPanel(61) +
  content(62+), momentum scroll pattern (CaptureSelect/Bounties).

AUDIO (100% synthesized - no files exist or may be added from the web)
- src/audio/audio.ts = WebAudio engine (typed SfxName switch + chip
  tracker); src/audio/tunes.ts = compositions as MIDI-number step
  arrays (16ths; null=rest; channels square/triangle/noise).
- Unlock: browsers gate audio behind a gesture; main.ts unlocks on
  first pointerdown, and TitleScene queues playMusic('home') EARLY so
  the tracker idles at t=0 and starts the instant the tap lands.
- Sound language: 'ui' click, 'tab' page-flip (top tabs), 'back' (a
  fourth lower, auto via BACK label in makeButton), 'loop' (rope
  pluck, heard constantly - keep soft), 'snap', 'release'. All mix
  volumes are constants in audio.ts.

  CaptureScene 'catches' + StageScene stageClear).

- Type assets: badges sprites/types/<Name>_46x15.png (key type-<Name>),

TUNING PHILOSOPHY (this is the entire feedback loop with the user)
- Every feel number is a NAMED CONSTANT at the top of its module:
  CaptureScene knobs, COOLDOWN_SCALE, damageRoll curve, stage speeds/
  ranges, classifier thresholds, mix volumes, FRONTIER_LEVELS. When
  playtest feedback arrives, the fix should be a one-line edit. When
  you add a system, expose its knobs the same way.
- Pure logic (geometry, LassoLine, stats, stage gen) stays in plain TS
  modules with no Phaser imports - reasoning without running depends
  on it.
